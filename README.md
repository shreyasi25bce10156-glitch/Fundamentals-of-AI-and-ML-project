# Student Performance Predictor

> **BTech AIML — Introductory Course Project**
> Binary classification model that predicts whether a student will **Pass** or **Fail**
> based on 9 academic, lifestyle, and background features.

---

## Quick Start

```bash
# 1. Install dependencies
pip install -r requirements.txt

# 2. Generate the dataset (creates student_performance.csv)
python generate_dataset.py

# 3. Train models & produce evaluation reports
python train_model.py

# 4. Launch the web interface
streamlit run app.py
```

All three scripts run **non-interactively** and exit with code 0 on success.

## Dataset

| Column | Type | Description |
|---|---|---|
| `study_hours_per_day` | float | Average daily self-study hours (0.5 – 10) |
| `attendance_percentage` | float | Class attendance rate (30 – 100 %) |
| `previous_gpa` | float | GPA from prior semester (0 – 10) |
| `sleep_hours_per_day` | float | Average nightly sleep (3 – 10 hrs) |
| `extracurricular_activities` | int | Participation in extracurriculars (0 = No, 1 = Yes) |
| `internet_usage_hours` | float | Non-academic internet use per day (0 – 8 hrs) |
| `parental_education_level` | int | Highest parental education (1 = HS, 2 = Bachelors, 3 = Masters, 4 = PhD) |
| `number_of_absences` | int | Total semester absences (0 – 30) |
| `assignment_completion_rate` | float | Percentage of assignments submitted (0 – 100 %) |
| **`performance`** | **str** | **Target: "Pass" or "Fail"** |

- **250 samples**, synthetically generated with a weighted scoring function + Gaussian noise.
- Expected class balance: ~62 – 68 % Pass, ~32 – 38 % Fail.

## ML Pipeline (`train_model.py`)

| Step | Description |
|---|---|
| **Load** | Read CSV, print shape / dtypes / stats / missing-value check |
| **EDA** | Class distribution bar chart, per-feature histograms split by class, Pearson correlation heatmap |
| **Preprocess** | `LabelEncoder` for target → {0, 1}; `StandardScaler` for features; 80/20 stratified split |
| **Train** | Four classifiers: Logistic Regression, Random Forest, SVM (RBF), KNN (k=5). Each evaluated with accuracy, precision, recall, F1, and 5-fold CV. |
| **Compare** | Grouped bar chart of metrics + horizontal CV bar chart |
| **Best Model** | Selected by highest F1 Score on test set. Detailed confusion matrix, classification report, ROC curve + AUC, and feature importance chart. |
| **Save** | `model_artifacts.pkl` containing the model, scaler, feature names, importances, class mapping, and performance metrics. |
| **Report** | Plain-text summary written to `reports/evaluation_report.txt`. |

## Web Interface (`app.py`)

- **Sidebar form** with 9 inputs (sliders + selectboxes) organised into Academic, Lifestyle, and Background sections.
- **Prediction page** shows a colour-coded Pass/Fail result box, probability bars, and a feature importance chart.
- **Landing page** (before prediction) displays model metrics, feature importance preview, and a "How It Works" explanation.
- Built with **Streamlit** — no external API calls, everything runs locally.

---

## Key Design Decisions

1. **Synthetic data with embedded logic** — A weighted scoring function creates a realistic signal-to-noise ratio so models can learn meaningful patterns while still making occasional errors.
2. **Four diverse classifiers** — Linear (Logistic Regression), tree-based (Random Forest), kernel (SVM), and instance-based (KNN) give a fair comparison for an introductory project.
3. **StandardScaler only where needed** — Distance-based and regularised models use scaled data; tree-based models use raw data. The `needs_scaling` flag in the artefacts ensures the app applies the same logic at inference time.
4. **Agg matplotlib backend** — All plots are generated headless-safe, so the pipeline works in CI/automated evaluation environments.
5. **Single artefact file** — `model_artifacts.pkl` bundles everything the app needs, avoiding scattered pickle files.

## Evaluation Notes

For automated evaluation:

```bash
pip install -r requirements.txt
python generate_dataset.py       # exit 0, prints sample count
python train_model.py            # exit 0, prints metrics, creates reports/
python -c "import joblib; a=joblib.load('model_artifacts.pkl'); print(a['model_name'])"
```

- `generate_dataset.py` creates `student_performance.csv` deterministically (seed = 42).
- `train_model.py` creates `reports/` with 7 PNGs + 1 TXT, and `model_artifacts.pkl`.
- `app.py` imports cleanly and loads artefacts via `@st.cache_resource`.
